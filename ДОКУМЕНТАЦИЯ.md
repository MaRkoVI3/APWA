## Въведение

APWA (AI Powered Wellbeing Assistant) е мобилно уеб приложение, предназначено за проследяване на емоционалното състояние на потребителя и предоставяне на персонализирани съвети за благополучие чрез изкуствен интелект.

### Основни цели

- Осигуряване на лесен и интуитивен интерфейс за проследяване на настроението
- Предоставяне на AI-базирани съвети за ментално здраве
- Насърчаване на здравословни навици чрез прости упражнения
- Демонстрация на интеграция между Python backend и JavaScript frontend

### Целева аудитория

Приложението е предназначено за потребители, които искат да следят своето емоционално състояние и да получават подкрепа в ежедневието си. Проектът е разработен като образователен пример за интеграция на AI в мобилни уеб приложения.

## Архитектура на системата

### Общ преглед

APWA следва архитектурата "клиент-сървър" с разделяне на отговорностите между frontend и backend компоненти.

```
┌─────────────────────────────────────────────────────────────┐
│                        Frontend                              │
│                    (index.html)                              │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐                   │
│  │  Home    │  │  Mood    │  │  Chat    │                   │
│  │  Screen  │  │  Screen  │  │  Screen  │                   │
│  └──────────┘  └──────────┘  └──────────┘                   │
└─────────────────────────────────────────────────────────────┘
                              │
                              │ HTTP/REST API
                              │
┌─────────────────────────────────────────────────────────────┐
│                        Backend                               │
│                    (FastAPI Server)                          │
│  ┌──────────────────────────────────────────────────────┐  │
│  │              API Endpoints                            │  │
│  │  GET  /           → Server status                    │  │
│  │  POST /mood       → Save mood entry                  │  │
│  │  GET  /mood/history → Get recent moods              │  │
│  │  POST /chat        → Get AI response                 │  │
│  └──────────────────────────────────────────────────────┘  │
│                              │                               │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │   Database   │  │   AI Module   │  │   CORS       │      │
│  │   (SQLite)   │  │  (Claude)     │  │   Config     │      │
│  └──────────────┘  └──────────────┘  └──────────────┘      │
└─────────────────────────────────────────────────────────────┘
```

### Компоненти

#### 1. Frontend (Клиент)

- **Единичен HTML файл**: Съдържа цялото UI и логика
- **Single Page Application (SPA)**: Навигация без презареждане
- **Responsive дизайн**: Адаптира се за мобилни устройства
- **PWA поддръжка**: Може да се инсталира на началния екран

#### 2. Backend (Сървър)

- **FastAPI**: Модерен Python web framework
- **RESTful API**: Стандартизирани endpoints за комуникация
- **CORS поддръжка**: Позволява cross-origin заявки
- **Uvicorn**: ASGI сървър за изпълнение на FastAPI

#### 3. База данни

- **SQLite**: Лека, файлова база данни
- **Една таблица**: `moods` за съхранение на записи
- **Автоматична инициализация**: Създава се при първо стартиране

#### 4. AI интеграция

- **Anthropic Claude API**: За генериране на отговори
- **System prompt**: Дефинира поведението на AI асистента
- **История на разговора**: Поддържа контекст между съобщения

## API документация

### Base URL

```
http://localhost:8000
```

### Endpoints

#### 1. GET /

Проверка дали сървърът работи.

**Response:**
```json
{
  "message": "APWA is running",
  "status": "ok"
}
```

#### 2. POST /mood

Записване на ново настроение.

**Request Body:**
```json
{
  "score": 4,
  "note": "Feeling pretty good today"
}
```

**Parameters:**
- `score` (integer, 1-5): Оценка на настроението
- `note` (string, optional): Допълнителна бележка

**Response:**
```json
{
  "success": true,
  "message": "Mood saved"
}
```

**Error Response (400):**
```json
{
  "detail": "Score must be between 1 and 5"
}
```

#### 3. GET /mood/history

Получаване на последните записи за настроение.

**Query Parameters:**
- `limit` (integer, optional): Брой записи (по подразбиране: 7)

**Response:**
```json
{
  "moods": [
    {
      "id": 1,
      "score": 4,
      "note": "Feeling pretty good today",
      "created_at": "2026-05-01 22:30:00"
    }
  ]
}
```

#### 4. POST /chat

Получаване на AI отговор.

**Request Body:**
```json
{
  "message": "I'm feeling anxious today",
  "history": []
}
```

**Parameters:**
- `message` (string, required): Съобщение на потребителя
- `history` (array, optional): История на разговора

**Response:**
```json
{
  "reply": "That sounds tough. Remember to be kind to yourself today...",
  "role": "assistant"
}
```

**Error Response (400):**
```json
{
  "detail": "Message cannot be empty"
}
```

## База данни

### Схема

#### Таблица: moods

| Колона | Тип | Описание |
|--------|-----|----------|
| id | INTEGER | Уникален идентификатор (PRIMARY KEY, AUTO INCREMENT) |
| score | INTEGER | Оценка на настроението (1-5) |
| note | TEXT | Допълнителна бележка (опционално) |
| created_at | TEXT | Дата и час на създаване |

### SQL схема

```sql
CREATE TABLE moods (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    score INTEGER NOT NULL,
    note TEXT,
    created_at TEXT NOT NULL
);
```

### Функции за достъп

#### init_db()

Инициализира базата данни и създава таблицата, ако не съществува.

#### save_mood(score, note)

Записва ново настроение в базата данни.

**Параметри:**
- `score` (int): Оценка 1-5
- `note` (str): Допълнителна бележка

**Връща:** ID на новия запис

#### get_recent_moods(limit=7)

Връща последните записи за настроение.

**Параметри:**
- `limit` (int): Максимален брой записи

**Връща:** Списък от речници с mood данни

## Frontend архитектура

### Структура на приложението

Frontend е реализиран като Single Page Application (SPA) в един HTML файл.

#### Екрани

1. **Home Screen** (`screen-home`)
   - Поздравление според часът на деня
   - Днешна дата
   - Карта с последно записано настроение
   - Карта с упражнение за дишане

2. **Mood Screen** (`screen-mood`)
   - 5 emoji бутони за избор на настроение
   - Текстово поле за бележка
   - Бутон за запазване

3. **Chat Screen** (`screen-chat`)
   - Заглавие със статус
   - Списък със съобщения
   - Поле за въвеждане
   - Предложени теми (chips)

### Навигация

Функцията `showScreen(screenId)` управлява превключването между екраните:

```javascript
function showScreen(screenId) {
    // Скрива всички екрани
    document.querySelectorAll('.screen').forEach(screen => {
        screen.classList.remove('active');
    });
    // Показва избрания екран
    document.getElementById(screenId).classList.add('active');
}
```

### API комуникация

Всички заявки към сървъра използват `fetch()` API:

```javascript
const API_BASE = 'http://localhost:8000';

async function saveMood() {
    const response = await fetch(`${API_BASE}/mood`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ score, note })
    });
    const data = await response.json();
    // Обработка на отговора
}
```

### Дизайн система

#### Цветове

| Цвят | Hex | Използване |
|------|-----|------------|
| Фон | #F5F0EB | Основен фон |
| Акцент 1 | #7C9D96 | Основен акцент |
| Акцент 2 | #E8A598 | Вторичен акцент |
| Текст | #2D2D2D | Основен текст |
| Мутекст | #8A8A8A | Вторичен текст |

#### Типография

- **Шрифт**: Nunito (Google Fonts)
- **Тежести**: 400 (regular), 700 (bold)
- **Размери**: 15px (body), 24px (headings)

#### Компоненти

- **Карти**: 18px border radius, лека сянка
- **Бутони**: 50px border radius, минимум 48px височина
- **Входни полета**: Същия стил като картите

## Сигурност

### Практики за сигурност

1. **Environment променливи**: API ключовете се съхраняват в `.env` файл, който не се включва в git

2. **CORS конфигурация**: В разработка се разрешават всички origins. В продукционна среда трябва да се ограничат.

3. **Валидация на входни данни**: Всички API заявки са валидирани чрез Pydantic модели.

4. **SQL инжекции**: Използване на параметризирани заявки предотвратява SQL инжекции.

5. **AI ограничения**: System prompt дефинира ясни граници за AI поведението.

### Препоръки за продукция

- Използване на HTTPS
- Ограничаване на CORS origins
- Добавяне на rate limiting
- Имплементиране на user authentication
- Редовни бекъпи на базата данни
